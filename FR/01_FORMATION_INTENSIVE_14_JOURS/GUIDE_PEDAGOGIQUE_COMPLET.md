# Guide Pédagogique Complet : Objectif 19.5

## Introduction
Ce guide est conçu pour vous accompagner dans votre préparation finale au BTS. Il synthétise les pièges à éviter et les standards de qualité Senior attendus par les examinateurs.

---

## 1. Stratégie pour la Partie Écrite (10 points)
- **Précision des termes** : Ne pas confondre HTML (structure) et CSS (style).
- **Composants URL** : Toujours citer le protocole, le domaine, le port et le chemin.
- **Marketing** : Savoir justifier pourquoi le digital supplante le traditionnel (coût, mesure, ciblage).

---

## 2. Stratégie pour le SQL (10 points)
- **Sujet 1 (Vols)** : Attention aux jointures triples (Pilote-Vol-Avion). Toujours utiliser `DISTINCT` si on demande "Les noms" sans doublons.
- **Sujet 2 (Ventes)** : Maîtriser les opérateurs de comparaison collective (`ALL`, `ANY`) et les fonctions d'agrégation dans les sous-requêtes.
- **Format** : Toujours terminer les requêtes par un point-virgule.

---

## 3. Stratégie pour le Web (10 points)
- **Sujet 1 (Maillots)** : L'accent est mis sur l'affichage dynamique via JS.
- **Sujet 2 (Scolaire)** : L'accent est mis sur la validation de formulaires complexes (Quartier, Téléphone).
- **Standard UI** : Utiliser Bootstrap pour une apparence professionnelle instantanée.

---

## 4. Stratégie pour l'UML & Java (10 points)
- **UML** : Respecter les flèches d'héritage (triangle vide) et d'association. Ne pas oublier les multiplicités (1..*, 0..1).
- **Java (Article)** : La gestion des exceptions est cruciale. L'utilisation de `protected` montre une maîtrise de l'héritage.
- **Java (Account)** : Les calculs financiers doivent être précis (utiliser `double`).

---

## Conclusion
Le succès repose sur la capacité à switcher entre les deux contextes (Aérien vs Commercial). Ce sprint vous entraîne à cette gymnastique mentale.
